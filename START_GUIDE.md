# 🍴 Culinary Conductor - Ultimate Setup Guide

Welcome to the Culinary Conductor full-stack application! This project uses **Next.js 14+**, **Neon PostgreSQL**, and **Tailwind CSS** to deliver a sleek, real-time restaurant ticketing system and a gorgeous "Ethereal Sovereign" themed UI.

---

## 🚀 1. Database Setup (Neon PostgreSQL)

We use [Neon.tech](https://neon.tech/) to run our highly scalable serverless Postgres schema.

**Step 1:** Create an account on Neon and initialize a new project.
**Step 2:** Copy your connection string into the `DATABASE_URL` environment variable inside of `.env`. 
*(We've placed a sample structure in your `.env` file.)*

**Step 3:** Go to your Neon Dashboard's "SQL Editor" tab and perform the migrations. You need to execute:
1. Copy the contents of `instructions/system.md` (Section 3: Database Schema) to build your tables (`users`, `products`, `orders`, `order_items`, `queue_state`, `analytics_events`).
2. Run those SQL schema commands.

**Step 4: Seed the Database!**
We just created a fresh `seed.sql` file in the root of your project!
Open `seed.sql`, copy the contents, and paste it directly into your Neon SQL Editor to inject placeholder items (burgers, tacos, and drinks!), a global queue configuration, and sample orders to immediately light up your admin dashboard.

---

## 🔑 2. Environment Setup

Update your `.env` file with your direct configurations:
- **`DATABASE_URL`**: Your Neon PostgreSQL string.
- **`JWT_SECRET`**: A random hash you generate for securely signing admin/customer authentication cookies.
- **`ADMIN_PASSWORD_HASH`**: The bcrypt hashed string of your chosen admin password. To test right out of the box, you can use: `$2a$10$YourHashGoesHereForAdmin123`. The email is `admin@aimiyskitchen.com`.
- **`IMAGEKIT_*`**: Add your ImageKit configuration keys here so they're in the environment whenever you write file-upload extensions.
- **`FAST2SMS_API_KEY`**: Insert your Fast2SMS credentials to bring SMS OTPs alive for customers!

---

## 💻 3. Running the Server

Run standard NPM commands from inside the `culinary-conductor` root directory:

```bash
# 1. Install all dependencies (if you haven't yet)
npm install

# 2. Start the dev server
npm run dev
```

---

## 📱 4. Using the App

### The Admin View (`http://localhost:3000/admin/login`)
- Log in using `admin@aimiyskitchen.com` and the raw equivalent of your `.env` password hash (if you injected our seed and didn't change `.env`, it expects `admin123`).
- **Dashboard:** Instantly see your metrics generated by the sample Seed dummy data.
- **Products:** Manage stock, change burger prices, or edit pictures.
- **Queue Control:** Easily the most powerful view. Push single clicks to *Advance Queue*, and watch Server-Sent Events (SSE) immediately pipe to your customers.

### The Customer View (`http://localhost:3000/login`)
- They get an industry-standard Fast2SMS OTP check. 
- During development (or if Fast2SMS keys are invalid), the backend will graciously send a debug OTP attached to your terminal logs or toast notifications!
- **Menu:** Beautiful grid.
- **Order Status:** Browse to `/order-status/2` to see your beautiful, dynamic ticket status screen that changes live when the admin touches "Advance Queue".
